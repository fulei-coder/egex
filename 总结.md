# U-arm 遥操作 RealMan (6DoF 先行 + 7DoF 预留) 总结与指南

**⚠️ 重要声明：当前采集只使用 ROS2 流程，不使用 ROS1/Noetic/catkin/rosrun。**

## 1. 当前已实现的功能
本套方案成功将 `LeRobot-Anything-U-Arm` (feat/ros2) 主手的遥操作能力接入到了 `ros2-lerobot-realman-vla` 采集与推理生态中，并达成了以下目标：
- **解耦的架构**：U-arm 作为纯粹的 ROS2 传感器输出 `/servo_angles`，主控制程序独立消费，互不干扰。
- **真实动作记录**：彻底废弃原始采集逻辑中使用下一帧状态作为行为 (`action = qpos[t+1]`) 的做法，改为记录真实下发的控制目标。
- **6/7自由度平滑过渡**：全链路数据（`qpos` 和 `action`）固定为 7 维格式，在当前阶段前 6 维执行机械臂关节控制，第 7 维安全占位（0.0）。后续增加夹爪时，不需要再改动模型特征结构。
- **硬件平滑与安全控制**：映射层内置 EMA（指数移动平均）滤波、跳变抑制、单步幅度限制，最大程度防止机械臂出现飞车或剧烈抖动。
- **推理期接口兼容**：推理脚本 (`inference.py`) 增加了夹爪执行隔离，即使模型输出了 7 维动作，目前也能过滤掉夹爪命令只执行臂部。

## 2. 文件结构与说明
- `configs/uarm_realman_map.yaml`：U-arm 映射关系的超参数中心（符号、重映射、缩放、死区、速度、是否启用夹爪）。
- `hardware/uarm_ros2_subscriber.py`：订阅 `/servo_angles` 提供非阻塞读取。
- `hardware/uarm_realman_mapper.py`：将 U-arm 角度转化为 RealMan 的目标角度，并处理夹爪占位及越界。
- `hardware/realman_teleop_controller.py`：以固定的 20Hz 循环提取最新指令执行 `rm_movej`，并将结果给到命令缓存。
- `scripts/collect_data_uarm.py`：专门用于 U-arm 遥操作采集的独立入口，保留了双相机记录和终端指令。
- `LeRobot-Anything-U-Arm-feat-ros2/src/uarm/scripts/Uarm_teleop/servo_reader.py`：修改后增加 `--ros-args -p serial_port:=/dev/tty...` 支持，避免串口漂移。

## 3. 使用方法与命令

### 3.1 启动 U-arm 主手节点
在新终端中，激活 ROS2，运行由于修改带来的可配串口节点：
```bash
source /opt/ros/humble/setup.bash
conda activate lerobot
# 确认 U-arm 插入的 USB 端口，例如 /dev/ttyUSB0
python3 /home/tony/lerobot-realman-vla/LeRobot-Anything-U-Arm-feat-ros2/src/uarm/scripts/Uarm_teleop/servo_reader.py \
    --ros-args -p serial_port:=/dev/ttyUSB0
```
你可以通过 `ros2 topic echo /servo_angles` 验证是否成功读到 7 维数据。

### 3.2 启动 DS87 腕部相机节点
在新终端中，按现有流程起 ROS2 Camera 驱动：
```bash
source /opt/ros/humble/setup.bash
source ~/ScepterSDK/3rd-PartyPlugin/ROS2/install/setup.bash
ros2 run ScepterROS scepter_camera
```

### 3.3 运行 U-arm 遥操作录制程序 (6DoF模式)
在你的项目根目录下打开终端：
```bash
source /opt/ros/humble/setup.bash
conda activate lerobot
cd /home/tony/lerobot-realman-vla

python3 scripts/collect_data_uarm.py \
    --arm-ip 192.168.1.18 \
    --save-dir data/raw_hdf5 \
    --task-name pick_cube \
    --fps 15
```
**按键操作：**
- `c`: 校准 U-arm 的零位（在此之前保持主手在想要的初始位置不要动）
- `w`: 启用遥控跟随（机械臂开始移动）
- `e`: 暂停遥控跟随
- `s`: 开始录制 HDF5 数据
- `d`: 停止录制并保存 Episode
- `q`: 退出程序

### 3.4 预演转换与推理
* **数据转换**：由于数据始终保持 7 维 `qpos` / `action`，你依然可以直接跑：
  ```bash
  python3 scripts/convert_to_lerobot.py --input-dir data/raw_hdf5 --output-dir data/pick_cube_15fps ...
  ```
* **模型推理**：在 6 自由度阶段推理（不执行夹爪）：
  ```bash
  python3 scripts/inference.py --model <your_model_dir> --freq 15
  ```
  如果未来接上夹爪了，推理时加上 `--state-with-gripper` 和 `--execute-gripper` 即可。

## 4. 硬件相关注意事项

1. **零位校准 (`v` 键) 至关重要**：U-arm 的绝对角度和 RealMan 差异极大。每次启用遥操作前，**一定**要把主手移动到一个舒适的起始位置，然后按 `v`，这会捕获此时主手的角度为“零点”。后续控制全靠相对于该零点的偏差 (`delta`) 驱动真机。
2. **夹爪的过渡与启用**：当前 `/servo_angles` 会发送 7 个值，但在代码中的 `configs/uarm_realman_map.yaml` 里设置了 `active_dof: 6` 和 `gripper.enabled: false`，所以你的抓取操作会被“屏蔽”并替换为 0.0 保存在文件里。以后如果增加了硬件夹爪，只需将 YAML 里 `enabled: true`，数据和控制信道就可以无缝切换过去。
3. **Modbus 被屏蔽**：为了防止在没有实体夹爪时触发 Modbus 死锁超时（导致主循环跌到 1Hz），我们在现在的设置下完全阻断了调用。不用担心没接夹爪出现通信严重延时的情况。
4. **安全保护阈值触发**：如果你发现动主手时，真机“卡住”不跟，通常是由于主手转动**太快或跳变过大**触发了 `jump_reject_deg: 20.0` 限幅。此时需要把主手退回到刚刚的位置，它就会重新吸合并继续跟随。
